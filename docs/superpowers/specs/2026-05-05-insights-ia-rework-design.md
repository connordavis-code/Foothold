## 1 ┬À Goal

Rework the `/insights` page from a single-card "show me the latest narrative" surface into a **latest-read + drilldown** surface where the AI narrative remains the hero, but is supplemented with verifiable structured numbers and entry points into the related detail pages (`/drift`, `/goals`, `/recurring`). Keep the editorial register from Phase 6.

This is a **UI + IA-only** change: no schema changes, no new server actions, no changes to how insights are generated, no changes to the cron, no changes to the dashboard `<InsightTeaserCard>`.

---

## 2 ┬À Locked decisions

| Decision | Choice |
|---|---|
| Page role | Latest read + drilldown ÔÇö narrative is the artifact, structured numbers are receipts, history is secondary but accessible |
| Layout register | Editorial single column, `max-w-3xl`, generous whitespace (current Phase 6 register, kept) |
| Supplements placement | Below the narrative (prose first; numbers as receipts) |
| History accessibility | Footer list at the bottom, click loads via `?week=YYYY-MM-DD` |
| Past-week supplements | Recompute live from current data ÔÇö no schema change, accept minor numerical drift if categorization changed after generation |
| Generate-button scope | Always generates for the **current** week regardless of `?week`. On past-week views, button collapses to a "Back to current week" link instead. |
| Invalid `?week` handling | Silent fallback to latest. No toast, no banner. |


### 4.1 HeaderBlock

- Eyebrow text: `TODAY` when viewing current/latest; `ARCHIVED` when `?week` is set.
- Title: `Insights` (unchanged).
- Subtitle: existing copy ("Weekly check-in on spending, goals, and recurring outflows ÔÇö generated by Claude.") unchanged.
- `<GenerateButton>` rendered top-right, three modes:
  - `generate` ÔÇö no insight exists yet. Label: `Generate insights`.
  - `regenerate` ÔÇö viewing latest. Label: `Regenerate`.

### 4.2 PastWeekBanner

- Shown only when `?week` resolves successfully.
- Single-line, low-emphasis (`bg-accent` or `bg-surface-sunken`).
- Contents: `ÔåÉ Back to current week` link ┬À `Viewing <week-of label>`.

### 4.3 NarrativeArticle

- Header inside the card: `WEEK OF <range>` eyebrow + `Generated <relative time>`. Stale chip (`┬À regenerate for current week`, amber) preserved unchanged for state 3 only.

### 4.4 ReceiptsSection

- Eyebrow: `WHAT CLAUDE SAW`.
- 2-up grid: `grid gap-3 sm:grid-cols-2` (matches the Drift "Elevated this week" grid).
- Tile recipe shared with Drift tiles: `rounded-card border border-border bg-surface-elevated p-5 sm:p-6`.
- Tiles render conditionally per the visibility rules (┬º7).
- Tile copy and link targets:
  - **Spending** (always): big number = `totalThisWeek`. Sub-line: `Ôåæ$Y vs 4-wk median` (or `Ôåô` / `~$X usual` if no delta). Below: top-3 category chips, each `<Category> ┬À $<total>`. No outbound link (data is fully shown locally).
  - **Drift** (conditional): heading `Drift`. Body: 1ÔÇô3 elevated category rows (`<Category> ┬À 2.1├ù usual ┬À $<current>`). Footer link: `See drift detail ÔåÆ` to `/drift`.
  - **Goals** (conditional): heading `Goals`. Body: stat `<onPace> of <active> on pace`. Below: 1 line for the most behind-pace goal. Footer link: `See goals ÔåÆ` to `/goals`.
  - **Recurring** (conditional): heading `Recurring`. Body: `$<thisWeek> hit this week ┬À $<monthly>/mo total`. Footer link: `See recurring ÔåÆ` to `/recurring`.
- If only Spending qualifies, grid degrades to `grid-cols-1` (single full-width tile).
- If no insight exists yet (state 1), section is not rendered.

### 4.5 EarlierWeeks

- Eyebrow: `EARLIER WEEKS`.
- Single bordered card, rows separated by `divide-y divide-border`.
- Each row: `<weekRange> "<firstSentence(narrative)>"` aligned left, `ÔåÆ` on right. The whole row is a `<Link href="?week=<weekStart>">`.
- Up to 6 rows. **No "See all" link in v1** ÔÇö the archive query simply caps at 6 (`LIMIT 6`). Re-add a "See all" target when archive density justifies it; an `/insights/archive` route is explicitly out of scope this round.
- When viewing a past week (`?week`), exclude that week's row from the list.
- If 0 rows remain after exclusion, the section is not rendered.

## 6 ┬À Data flow

### 6.1 Page-load sequence

```
1. const session = await auth(); if (!session?.user) return null;
2. const params = await searchParams;                       // Next 14 RSC async
3. const weekKey = resolveWeekParam(params.week);           // shape-only validation
4. const [insight, archive] = await Promise.all([
     weekKey
       ? getInsightForWeek(userId, weekKey)
       : getLatestInsight(userId),
     getInsightsForArchive(userId, 6),
   ]);
5. const supplements = insight
     ? await getInsightSupplements(userId, insight.weekStart, insight.weekEnd)
     : null;
```

Notes:
- The archive query and the displayed-insight query are separate. The archive query truncates `narrative` to 400 chars for first-sentence previews; the displayed insight always comes from a full-row query (`getLatestInsight` or `getInsightForWeek`). Re-using archive rows for the article body would render truncated prose ÔÇö keep the two paths separate.
- Step 4 runs in parallel: archive + insight read share no dependency. Step 5 waits on step 4 because supplements scope on `(weekStart, weekEnd)`.
- `resolveWeekParam` validates shape only (well-formed `YYYY-MM-DD`, not in the future). It does NOT check existence ÔÇö that's `getInsightForWeek`'s job, which returns `null` for misses. The page treats `insight === null` after a `?week=` request as state 5 (silent fallback) ÔÇö but at that point latest is no longer in scope; we render state 1 (Pristine) **only if** the user has zero insights total. If the user has insights but `?week=` missed, the cleanest behavior is: re-issue `getLatestInsight` to render state 2/3. Plan should include this guard or eliminate it by ensuring the resolver consults the archive list before deciding to fetch (the cheaper path). Recommendation: keep the resolver pure (shape-only), let the page do a second `getLatestInsight` if `?week=` produced a null ÔÇö costs one extra query in the rare bad-link case.

### 6.2 `getInsightSupplements` shape

```ts
export type InsightSupplements = {
  spending: {
    totalThisWeek: number;
    deltaVsBaseline: number | null;
    topCategories: { category: string; total: number }[];  // top 3
  };
  drift: {
    elevated: { category: string; ratio: number; currentTotal: number; baselineWeekly: number }[];
    hasBaseline: boolean;
  };
  goals: {
    activeCount: number;
    onPaceCount: number;
    notable: { name: string; pacePct: number; type: GoalType }[];  // top 1ÔÇô2 most behind
  };
  recurring: {
    hitThisWeekCount: number;
    hitThisWeekTotal: number;
    monthlyTotal: number;
  };
};
```

Implementation:
- Spending ÔÇö same SQL pattern as `collectSnapshot.spending`. Top-3 from `byCategoryThisWeek`.
- Drift ÔÇö `getDriftAnalysis` already returns `currentlyElevated` for the *most recent* week. For past-week views, we need elevated detection for an arbitrary week. Two paths: (a) parameterize `getDriftAnalysis` with an `endAnchor` arg; (b) write a smaller `getElevatedForWeek(userId, weekStart, weekEnd)`. Plan should pick one ÔÇö leaning (a) since `buildWeekWindows` already supports it.
- Goals ÔÇö `getGoalsWithProgress` already returns the data; filter to active and compute `pacePct` per goal. `notable` = the 1ÔÇô2 with the worst pace.
- Recurring ÔÇö reuse `getRecurringStreams`, filter to outflows hitting in-week.

### 6.3 Generate button behavior

- `mode="generate"` and `mode="regenerate"`: existing action `generateInsightAction()` (no-arg, current-week) is invoked. On success, the button's existing `revalidatePath('/insights')` is supplemented with `router.push('/insights')` to strip any `?week=`.
- `mode="back"`: rendered as `<Link href="/insights">`, no client action.

### 6.4 Invalid `?week`

- `resolveWeekParam` returns `null` when the param is undefined, malformed, in the future, or doesn't match any archive row AND `getInsightForWeek` (fallback) finds nothing.
- The page treats `null` as "show latest". No banner, no toast, no redirect ÔÇö the URL stays `?week=ÔÇª` but the page renders state 2 or 3.

## 7 ┬À States

| # | State | Trigger | Renders |
|---|---|---|---|
| 1 | Pristine | No `insight` rows for user | EmptyState card. No archive footer. Button: `generate`. |
| 2 | Current, fresh | Latest exists; `weekEnd === yesterday`; no `?week` | TODAY ┬À article (no stale chip) ┬À receipts ┬À archive (if any prior). Button: `regenerate`. |
| 3 | Current, stale | Latest exists; `weekEnd !== yesterday`; no `?week` | TODAY ┬À article (with amber stale chip) ┬À receipts for latest's week ┬À archive. Button: `regenerate`. |
| 4 | Past week | `?week` resolves | ARCHIVED ┬À banner ┬À article (no stale chip) ┬À receipts for that week ┬À archive (excluding viewed). Button: `back`. |
| 5 | Invalid `?week` | Param parses but no match | Silent fallback to state 2 or 3. |
| 6 | Generating | Action in flight | Button shows spinner; rest of page mounted. |
| 7 | Generation error | Action throws | Inline error chip below button (existing pattern). |
| 8 | Snapshot empty | `collectSnapshot` returns `isEmpty:true` | Surfaces as state 7 with "Not enough data this week to summarize". |

### Receipts visibility rules (states 2/3/4)

- `<SpendingTile>` ÔÇö always.
- `<DriftTile>` ÔÇö `elevated.length > 0 && hasBaseline`.
- `<GoalsTile>` ÔÇö `activeCount > 0`.
- `<RecurringTile>` ÔÇö `monthlyTotal > 0`.
- If only Spending qualifies, grid is `grid-cols-1`.

### Archive footer visibility

- States 1, 5: never.
- States 2, 3: iff archive has ÔëÑ1 prior beyond latest.
- State 4: iff anything remains after excluding the viewed week.

### 10.1 RSC searchParams

### 10.2 Drift baseline parameterization

`getDriftAnalysis` currently anchors on yesterday. For state 4, supplements need elevation detection for an arbitrary week. Plan should:
- Add an optional `endAnchor: string` parameter to `getDriftAnalysis` (defaulting to yesterday), OR
- Introduce a smaller `getElevatedForWeek(userId, weekEnd)` that reuses `buildWeekWindows`.

Recommend the parameter ÔÇö surface area stays smaller and `/drift` itself can keep its current call shape via the default.

### 10.3 First-sentence extract

`firstSentence` currently lives inline in `src/components/dashboard/insight-teaser-card.tsx`. Move to `src/lib/utils/first-sentence.ts`; update both teaser card and new earlier-weeks list to import from there. Keep behavior identical to avoid changing the dashboard preview.

### 10.4 Empty-state copy refresh (light)

Existing EmptyState copy:
> "Click 'Generate insights' to have Claude summarize the last 7 days of spending, goal progress, and recurring outflows. Each generation costs a fraction of a cent and takes a few seconds."

Lightly refresh to mention the receipts:
> "Generate to see Claude's read on the last 7 days ÔÇö spending, goal pace, recurring outflows ÔÇö alongside the underlying numbers."

Optional. If implementation discovers the new copy reads worse in context, keep the existing.

### 10.5 Archive query

```sql
SELECT week_start, week_end, generated_at,
       SUBSTRING(narrative FROM 1 FOR 400) AS narrative
FROM insight
WHERE user_id = $1
ORDER BY week_start DESC
LIMIT 6
```

Truncating in SQL avoids loading full bodies just to extract a first sentence. Return type omits the truncation detail at the consumer layer ÔÇö type stays `Insight`-shaped.

### 10.6 Loading skeleton

Renders header skeleton + narrative card skeleton (kept) + 4-tile receipts skeleton (over-show is fine) + no archive skeleton. Path stays `src/app/(app)/insights/loading.tsx`.
