### Foothold token gotchas (from R.2 + R.3.x experience)

Documented in CLAUDE.md > **Foothold Redesign milestone**:

- Complete-color tokens (`--surface`, `--text-*`, `--semantic-*`,
  `--dot-halo`) work as raw `var()` and as Tailwind arbitrary values
  `bg-[--surface]`, `text-[--text-2]`
- Shadcn-mapped HSL fragments (`--accent`, `--positive`, `--background`)
  need `hsl(var(--x))` wrapping OR Tailwind config mapping
  (`bg-accent` works via `tailwind.config.ts`)
- `bg-[--accent]` silently generates invalid CSS — DON'T use the
  arbitrary-value syntax on HSL fragments
- `text-positive` and `text-destructive` work via Tailwind config
