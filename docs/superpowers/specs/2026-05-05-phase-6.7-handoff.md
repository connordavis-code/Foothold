### 2. /transactions select-all checkbox + sonner-with-undo on bulk re-categorize

Two related operator gaps from the critique:

**(a) Select-all-visible checkbox.** The header column in `operator-table.tsx`
is currently `<Th className="w-[36px] text-center" aria-label="Select" />` —
empty. Selecting 50 rows requires shift-click first→last. Add a tri-state
checkbox to the header.

States:
- **None selected** — empty checkbox, click → select all visible rows
- **All visible selected** — checked, click → deselect all
- **Some selected** (≥1 but not all) — indeterminate state (`aria-checked="mixed"`), click → select all

Visible = the rows currently rendered (paginated set). Cross-page select-all
is OUT OF SCOPE for this phase — would need a different mental model.

**(b) Sonner-with-undo on bulk re-categorize.** Currently `BulkActionBar`
fires `updateTransactionCategoriesAction` and shows a sonner success toast
with no undo affordance. Add an Undo action:

```ts
toast.success(`Re-categorized 23 transactions as "Coffee".`, {
  action: {
    label: 'Undo',
    onClick: () => undoBulk(),
  },
});
```

The undo path needs to restore the per-row prior categories. Two strategies:

1. **Client-side capture.** Before invoking the action, snapshot
   `Map<txId, prior categoryOverrideId | null>` from the rendered rows.
   On Undo, call `updateTransactionCategoriesAction` once per distinct
   prior category (group by category, batch within group). N round-trips
   bounded by distinct prior categories, typically ≤5.
2. **Server-side snapshot.** Add a new query that returns + restores
   pre-state in one shot. More work; defer.

**Strategy 1 is the call.** Simpler, no server changes.

**Files:**
- Edit: `src/components/transactions/operator-table.tsx` — add header checkbox
- Edit: `src/components/transactions/operator-shell.tsx` — wire `selectAllVisible` / `clearAllVisible` callbacks; expose row data for snapshotting
- Edit: `src/components/transactions/bulk-action-bar.tsx` — capture pre-state, add sonner undo action, restore via grouped action calls
- No changes to `src/lib/transactions/actions.ts` needed if grouping is done client-side

**Acceptance:**
- Header checkbox: tri-state, visually clear (use `data-state="indeterminate"` styling)
- Click on indeterminate header → selects all visible (most expected behavior)
- Bulk re-categorize toast has an Undo button
- Undo restores per-row prior categories correctly
- Undo button disappears after toast auto-dismisses (sonner handles this; ~5s default)
